## 2. Startup connection-priority probe (the early win — read-only, strictly additive)

- [x] 2.1 Add a scale-streaming-confirmed signal/flag: `WeightProcessor` (or the scale path) reports when the scale has delivered a healthy run of weight notifications at expected cadence (known-good baseline). This is the precondition for the probe — never fire on bare scale-connect.
- [x] 2.2 Add a probe controller (in/near `QtScaleBleTransport` / the DE1 transport boundary) that, gated on `scale-streaming-confirmed ∧ DE1-connected ∧ machine-idle ∧ not-yet-probed-this-connect ∧ latch-not-set`, runs a **bounded read-only** DE1 burst: MMR block reads (`a005`) over the known-safe address range + a read-poll loop of safe readable characteristics only (`a001`, `a00a`, `a00e`, `a011`). It MUST NOT write or poll any state-affecting/DANGER char (`a002`/`a006`/`a009`/`a00b`/`a00f`/`a010`/`a012`), MUST enqueue via the normal BLE command queue (respect 50 ms write spacing rule; reads enqueued normally), and MUST run at most once per scale connect.
- [x] 2.3 Set `WeightProcessor::m_probeActive` for the probe's duration (cross-thread → explicit `Qt::QueuedConnection`) so a probe-provoked scale-feed stall drives the *existing* `scaleFeedStalled` → backoff → reconnect-at-BALANCED path with zero new mechanism. Clear it when the probe ends.
- [x] 2.4 Yield-on-shot: if an espresso cycle starts while the probe is active, end the probe immediately (clear `m_probeActive`, stop the burst) without itself triggering a backoff; normal preheat/extraction liveness governs from there.
- [x] 2.5 Observability (D6): log a distinct probe-start and probe-end line at the existing WARN marker level, and ensure a probe-provoked backoff still emits the existing `Scale connection-priority BACKOFF triggered …` marker — so a single reporter debug.log shows definitively whether the probe ran, whether it provoked, and the outcome.
- [x] 2.6 Internal kill-switch: a non-UI compile/config flag to disable the probe (cheap insurance if a field issue emerges); default enabled.
- [x] 2.7 Tests: capable-path (probe runs, no stall, no trip, HIGH retained); weak-path (probe window active + simulated stall → existing backoff fires at idle); probe does not start before streaming-confirmed; probe yields when a cycle starts mid-probe; probe runs at most once per connect.
